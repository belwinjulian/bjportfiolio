  <action>
Create `components/Navigation.tsx` as a client component (`'use client'`).

**Navigation structure:**
- Sticky positioning: `sticky top-0 z-50`
- Frosted glass effect: `bg-background/80 backdrop-blur-md border-b border-border`
- Container: `max-w-6xl mx-auto px-4 sm:px-6 lg:px-8`
- Height: `py-4` for comfortable touch targets

**Desktop layout (md and above):**
- Left: Name/logo text "BJ" or "Belwin Julian" as a link that scrolls to top — use `text-lg font-bold text-foreground`
- Right: Navigation links in a horizontal `flex gap-8` list
- Links: "About", "Projects", "Resume", "Contact" — these are placeholder section names for future phases. Use `<a href="#about">` pattern so smooth scroll works via CSS `scroll-behavior: smooth`
- Link styles: `text-foreground/70 hover:text-foreground transition-colors duration-200`

**Mobile layout (below md):**
- Left: Same name/logo
- Right: Hamburger button (three horizontal lines icon using div/span elements, NOT an external icon library)
- When hamburger is clicked, toggle a mobile menu panel below the nav bar
- Mobile menu: `flex flex-col gap-4 px-4 pb-4` with the same links
- Use `useState` for open/close state
- Add `aria-label="Toggle navigation menu"` to hamburger button
- Add `aria-expanded` attribute reflecting menu state

**Smooth scroll behavior:**
- Use `<a href="#sectionId">` anchor links — CSS `scroll-behavior: smooth` in globals.css handles the animation
- Do NOT use `scrollIntoView` JavaScript — the CSS approach is more accessible and respects `prefers-reduced-motion`
- Close mobile menu after clicking a link (set state to false in onClick handler)

**Scroll offset consideration:**
- Since nav is sticky, scrolled-to sections will be partially hidden behind it
- Add `scroll-mt-20` to section elements (handled in Task 3 when wiring the page)

**Do NOT:**
- Install any icon library (use CSS/HTML for hamburger icon)
- Add active link highlighting (defer to Phase 4 polish)
- Add scroll-based background opacity changes (defer to Phase 4)
